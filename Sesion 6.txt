Estructuras de datos dinámicas
Son estructuras que pueden cambiar de tamaño durante la ejecución del programa. Permiten agregar o eliminar datos según sea necesario, ofreciendo mayor flexibilidad y eficiencia.
Tipo Abstracto de Datos (TAD)
Es una descripción de un conjunto de datos y de las operaciones que se pueden realizar sobre ellos. Se enfoca en qué hace la estructura y no en cómo está implementada.
Conceptos clave de un TAD


Dominio de valores: conjunto de valores que puede almacenar.


Operaciones: acciones que se pueden realizar, como insertar, eliminar o buscar.


Especificación formal: definición precisa del comportamiento de las operaciones.


Implementación: código que hace funcionar el TAD en un lenguaje de programación.


Especificación informal de un TAD
Describe las operaciones usando lenguaje natural.
Ejemplo: una pila permite insertar y eliminar elementos siguiendo el orden LIFO (el último en entrar es el primero en salir).
Especificación formal de un TAD
Usa notación matemática o lógica para definir exactamente el comportamiento de las operaciones.
Elementos importantes


Precondición: condición que debe cumplirse antes de ejecutar una operación.


Postcondición: resultado que debe cumplirse después de la operación.


Invariante: propiedad que siempre se mantiene en la estructura.


Ejemplo de una pila (stack)
Operaciones principales


push(x): inserta un elemento en la parte superior.


pop(): elimina y devuelve el elemento superior.


top(): muestra el elemento superior sin eliminarlo.


isEmpty(): verifica si la pila está vacía.


Reglas de la pila


Para usar pop() o top(), la pila no debe estar vacía.


Después de push(x), el elemento x queda en la parte superior.


Los elementos mantienen el orden de inserción tipo LIFO.


Implementación de un TAD
Consiste en programar las operaciones del TAD usando estructuras de datos adecuadas.
Formas comunes de implementar una pila


Con arreglos: usando índices para controlar la parte superior.


Con listas enlazadas: usando nodos conectados entre sí.


Utilidad de los TAD y estructuras dinámicas


Mejoran la organización y manejo de datos.


Facilitan programas más escalables y reutilizables.


Permiten usar memoria de forma más flexible y eficiente.
